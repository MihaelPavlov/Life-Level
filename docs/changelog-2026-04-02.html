What Changed

Fix Strava OAuth — 3 Bugs Resolved

  • Missing redirect_uri in token exchange
    StravaOAuthService.cs — Added "redirect_uri" = req.RedirectUri to the Strava token exchange form. Without it Strava can reject the exchange with an "invalid_grant" error.
  • Double deep-link firing on cold launch
    main_shell.dart — Added _oauthCallbackHandled bool flag. Both getInitialLink() and uriLinkStream fire on a cold-launch redirect; the guard ensures the OAuth code is only consumed once.
  • App navigating away from Integrations after OAuth redirect
    main_shell.dart — Removed Navigator.push(IntegrationsScreen) from _handleStravaCallback. The existing Integrations screen auto-rebuilds via Riverpod when the provider state changes to isStravaConnected: true.
  • Unique-index crash when re-connecting same Strava account
    StravaOAuthService.cs — Changed lookup order: query by StravaAthleteId first. This prevents a DB unique constraint violation when the same Strava athlete reconnects or switches LifeLevel accounts.

FeatureBackend Items System — Categories & Drop Rules

  • Added ItemCategory enum: Tracker, Clothing, Footwear, Accessory
  • Expanded catalog from 8 → 21 items, organised by category, rarity Common → Legendary
  • New Strava Sync Badge item (Rare Tracker) — awarded automatically when Strava is connected
  • Added AcquisitionTrigger + ItemDropRule entity — items can now drop on events (level-up, Strava connect, distance milestones)
  • ItemGrantService.cs — new service that evaluates drop rules and grants items; triggered after activity logging and level-up
  • DB migrations applied: AddItemCategoryAndDropRules, AddItemDropRuleTriggerParametersLength

UI Profile Screen Overhaul

  • Settings Sheet — The gear icon now opens a bottom sheet with two options: Integrations (navigates to integrations screen) and Logout (red, clears token and returns to login).
  • Achievements Tab — Replaced placeholder stub with a real AchievementsTab widget.
  • Admin Tab — Shown only for Admin-role users; links to the admin web panel and map editor inside a WebView.
  • Admin Nav Panel — Profile header shows admin-only shortcuts when the JWT role is Admin.
  • Gear Bonus Chips on Stats — Each stat card now shows a green +N chip when equipped gear adds a bonus. profile_stat_metadata.dart, profile_overview_tab.dart
  • Equipment unequip refreshes character profile — equipment_tab.dart / inventory_tab.dart both call characterProfileProvider.notifier.refresh() after equip/unequip so stats update instantly.

UI Inventory Tab Improvements

  • Warning banner appears when inventory is over 70 % full: "Level up to unlock more inventory slots"
  • Item detail sheet shows +N XP from gear badge when gear XP bonus is active
