Tutorial — Floating Bubble Coach Marks + Tutorials Hub

Variant B is the chosen design. Each coach mark is a floating speech-bubble tethered to its target with a tail; the rest of the screen is dimmed and the target gets a pulsing accent ring. The tutorial is framed as an in-world quest — first completion awards +500 XP and the Novice title. It's skippable at every step and replayable anytime from Profile → Settings → Tutorials, split into five topic tutorials that can each be replayed independently.

Variant A · Bottom callout + spotlight

Callout docked to the bottom with step counter and actions. Great on small targets; reads as a "system" tooltip rather than in-world dialogue.
FLOW
8 states — Step 0 (intro full-screen) → 6 floating-bubble coach marks → Step 7 (outro full-screen). Total reward on first completion: +500 XP + Novice Adventurer title.
GATE
Step 4 is action-gated — the Next button is replaced by a disabled "Waiting…" pill until a real activity is logged. The ActivityService advances the tutorial server-side on first log.
HUB
Tutorials hub lives in Profile → Settings → Tutorials — "Play all" runs the entire flow again (intro + 6 bubbles + outro) with zero rewards. Each topic row runs only its own bubble(s) and updates its ✓ status via tutorialTopicsSeen bitmask.
TECH
No new Flutter dependency — bubble body is a plain Container; tail is a CustomPainter triangle (~30 LOC). Placement picker reads each target's GlobalKey rect per frame; applies the 4-case rule: above / below / above-right-offset / above-FAB.
SKIP
Skip confirmation reminds users they can replay anytime. Once skipped, TutorialStep = -1 and the auto-start does not fire again. The hub still works (skipping never breaks replay access).
