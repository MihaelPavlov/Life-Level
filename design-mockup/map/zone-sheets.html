Implementation contract

  • Header emoji tint is type-driven: teal=entry, green=standard/branch, purple=crossroads (rotated 45°), red=boss. Status overrides the border colour (green when completed, red glow when boss + active).
  • Status pill is state-driven; always present below the region line.
  • Chips row is three columns (Level req · Distance · XP) for every type except Crossroads — which swaps chips for the dashed purple note.
  • CTA colour by action: orange=set/re-affirm, red=fight boss, purple=open choice sheet, green=disabled-done, blue=disabled-here, grey=locked.
  • Boss · Active is the only CTA that performs a destructive call (completeZone) directly from the sheet; all other actions either set destination (idempotent) or open another sheet.
  • Crossroads and Branch states rely on branchOf + pathChoices already carried by RegionDetailDto. No new backend fields needed.
  • Chest is a one-shot zone — tap 🎁 Open chest when arrived, grants XP, flips to the green opened state permanently. No cooldown.
  • Dungeon is a multi-floor cross-training gate. Three floors, each with a distinct ActivityType + target. Floors unlock sequentially. Setting a destination outside the dungeon with incomplete floors forfeits them permanently (Abandoned state). Activity logging auto-credits toward the active floor when the logged ActivityType matches.
